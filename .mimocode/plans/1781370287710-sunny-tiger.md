# Auto-Save & Temporary Save for Exam Attempts

## Problem
Currently, exam answers are only persisted when the user submits. If they exit mid-exam (back button, cancel dialog, or app background), all answers are lost. There is no draft/resume functionality.

## Solution
1. **Auto-save** after every answer change (persist draft to Room DB)
2. **"Save & Exit" button** in the exam top bar
3. **Draft resume** — show "Continue" button on dashboard and exam detail when a draft exists
4. **Updated cancel dialog** — offer to save before exiting

## Key Design Decisions

- **Draft = ExamAttempt with `isDraft = true`**: Reuses existing table, no new entities needed
- **Grading fields = 0 for drafts**: `correctCount`, `incorrectCount`, etc. are 0 until submission
- **`@Insert(onConflict = REPLACE)`**: Existing DAO method handles both insert and update (upsert by primary key)
- **Debounce auto-save**: Use `snapshotFlow` on `activeAnswers` with a short delay to avoid excessive DB writes
- **Draft cleanup**: Draft is deleted when exam is submitted or when user explicitly starts a new attempt
- **`getAttemptsForExam` excludes drafts**: So results/history lists only show completed attempts

## Verification

1. Start an exam, answer a few questions, press back → draft saved
2. Return to dashboard → "Continue" badge visible on exam card
3. Tap "Continue" → exam resumes with saved answers
4. During exam, tap "Save & Exit" → returns to exam detail, draft preserved
5. Submit exam → draft deleted, attempt appears in history
6. Start exam with existing draft → prompt to continue or start fresh
7. Auto-save: answer questions, force-close app, reopen → answers restored
